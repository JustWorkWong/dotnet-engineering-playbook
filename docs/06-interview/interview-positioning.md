# 面试定位与训练方法

## 1. 面试不是主线，是输出层

这个仓库里，面试不应该和生产实践分家。

更稳的理解是：

```text
底层原理
  -> 工程实践
  -> 架构 trade-off
  -> 线上问题
  -> 面试表达
```

如果直接从最后一层开始，只会得到零散八股；如果从前四层往下压，面试回答才会有厚度。

## 2. 每个主题至少练三种回答

### 1 分钟

- 先讲结论
- 再讲适用场景
- 最后讲一个关键 trade-off

### 3 分钟

- 讲核心机制
- 讲为什么这么设计
- 讲常见坑和兜底方式

### 10 分钟

- 讲完整链路
- 讲失败模式
- 讲为什么不选别的方案
- 讲一个真实或模拟案例

## 3. 面试官真正想看什么

大多数深挖题，最后都在看下面四件事：

1. 你能不能把问题说清楚
2. 你知不知道方案代价
3. 你能不能识别失败模式
4. 你有没有工程闭环意识

所以同一个问题，不要只准备“是什么”，还要准备：

- 为什么
- 什么时候别用
- 出事后怎么查
- 如果重来一次怎么设计

## 4. 建议训练闭环

```text
专题文档
  -> 最小 demo
  -> 失败案例
  -> 1/3/10 分钟回答
  -> 追问复盘
```

## 5. 当前仓库里的推荐训练主题

1. `.NET` 底座：异步、线程池、GC、请求管线
2. CAP：最终一致性、幂等、顺序性、补偿
3. OpenTelemetry：Trace、Metric、Log、采样、上下文传播
4. 埋点：业务事件、审计、AI 成本分析
5. AI 协作：`Claude Code / Codex / ChatGPT` 的分工和边界

## 6. 一条硬标准

如果一个回答离开具体场景就讲不下去，说明它更像背诵，不像工程判断。
